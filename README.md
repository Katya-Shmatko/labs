# Лабораторні роботи 2-6, які реалізують BlockProcessor з SQL-бекендом
Цей репозиторій містить систему `BlockProcessor` — програму на Python для обробки, валідації та збереження даних про "blocks" та "votes" користувачів. Проект побудований з використанням бази даних SQLite. 
## Структура проекту та етапи розробки:
* Лабораторна робота 2 (`lab2.py`, `lab2.csv`): Читання даних із CSV-файлу. Початкові класи `Block` та `Vote` за допомогою `dataclasses`, додано перевірку шістнадцяткового формату (HEX).
* Лабораторна робота 3 (`lab3.py`): Створено структуру таблиць SQLite (`BLOCKS`, `SOURCES`, `VOTES`, `PERSONS`) та написано скрипт для їх наповнення.
* Лабораторна робота 4 (`lab4.py`): Написано методи класу `@classmethod` для витягування записів із бази даних SQLite та перетворення їх на об'єкти Python.
* Лабораторна 5 (`BlockProcessor.lab5.py`, `updater.lab5.py`): Реалізовано два окремі скрипти, які пов'язані між собою через базу даних і працюють паралельно. Перший скрипт приймає дані від користувача та записує подію в таблицю `event_stream` та ставить статус `processed = 1` . Другий скрипт нескінченно перевіряєнові записи до таблиці `event_stream` та змінює статус на `processed = 0`.
* Лабораторна 6 (`lab6.py`, `test_lab6.py`): Впровадження типізації та валідації даних за допомогою бібліотеки `pydantic`. Написано набір тестів (`pytest`) для перевірки обмежень.
